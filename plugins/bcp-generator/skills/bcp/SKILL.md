# BCP自動生成スキル

企業WebサイトのURL**または社名**からBCP（事業継続計画）を自動生成し、HTMLとして出力します。

## スキルの動作

URLまたは社名を受け取ったら、以下の7ステップを**自律的に**完走してください。
**ユーザーへの追加質問は原則ゼロ**（社名から候補を絞り込む場合を除く）。情報不足でも `[要確認]` で埋めて進行すること。

---

## Step 1: 入力判定＋情報収集（多重ソース）

`$ARGUMENTS` から入力を取得する。入力がなければユーザーに1回だけ聞く。

### Step 1a: 入力の種別判定

入力がURLか社名かを判定する:
- **URL判定**: `http://` `https://` で始まる、またはドメイン形式（`xxx.co.jp` `xxx.com` 等）→ URLとして扱う → **Step 1c** へ
- **それ以外** → 社名として扱う → **Step 1b** へ

### Step 1b: 社名からの企業特定（社名入力の場合のみ）

1. `WebSearch` で「{社名} 公式サイト」を検索
2. 検索結果から**企業の公式サイト候補**を抽出する（上位5件程度）
3. 以下の基準で候補を判定:
   - 企業名が入力と一致・類似している
   - 公式サイトである（コーポレートサイト、`co.jp` ドメイン等）
   - 求人サイト・口コミサイト・ニュース記事は除外

**候補が1社に明確に絞れる場合:**
- そのURLを採用し、**Step 1c** へ進む

**候補が複数ある場合（同名・類似名の企業が存在）:**
- ユーザーに候補リストを提示して選択を求める（**これは許容される質問**）
- 提示形式:
  ```
  「{社名}」に該当する企業が複数見つかりました。どちらの企業ですか？

  1. {企業名A} - {業種/所在地} - {URL}
  2. {企業名B} - {業種/所在地} - {URL}
  3. {企業名C} - {業種/所在地} - {URL}
  ```
- ユーザーの選択後、そのURLで **Step 1c** へ進む

**候補が見つからない場合:**
- `WebSearch` で「{社名} 会社概要」「{社名} 企業情報」と追加検索
- それでも見つからない → WebSearch結果から得られる情報だけで **Step 1c** へ進む（URL不明のまま続行）

### Step 1c: 企業情報の収集

**以下を並行実行:**

1. `WebFetch` で企業サイトを読み取る（URLがある場合）
   - まずトップページ
   - 次に「会社概要」「about」「company」等のリンクを探してWebFetch
   - 「アクセス」「access」ページも探してWebFetch
2. サイトから会社名を特定したら `WebSearch` で「{会社名} 会社概要 事業内容 所在地」を検索

**抽出する情報:**
- 会社名
- 業種（推定含む）
- 事業内容・主要サービス/製品
- 所在地（住所）
- 従業員数（わかれば）
- 主要取引先（わかれば）
- ※売上・年商・収益などの金額情報は収集・記載しない

**自己回復ルール:**
- WebFetch失敗 → WebSearch結果のみで続行
- 住所不明 → WebSearchで追加検索「{会社名} 住所 アクセス」
- 最低限「業種の推定」と「住所の一部」があればStep 2へ進む
- 全情報源が失敗 → ユーザーに「業種と所在地」を1回だけ聞く

## Step 2: 住所→座標変換

```bash
python3 ${CLAUDE_SKILL_DIR}/../../scripts/geocode.py "{住所}"
```

出力はJSON: `{"lat": 35.xxx, "lng": 139.xxx, "label": "..."}`

**フォールバック:**

- それでも失敗 → WebSearchで「{会社名} 住所」を検索して再試行
- 最終手段: 都道府県庁所在地の座標で代替（`[所在地要確認]` 付記）

## Step 3: ハザードデータ収集（並列実行必須）

**重要: 以下の3a/3b/3dは3つの並列Bash呼び出しとして同時実行すること。3c/3eのWebSearchも同時に開始する。**
**合計5系統を同時並列で実行し、全完了を待つ。逐次実行は禁止。**

### 並列グループA: 3つのBashコマンドを同時実行

以下の3コマンドを **1つのメッセージ内で3つの並列Bash呼び出し** として実行する:

**3a. ハザードマップタイル分析:**
```bash
python3 ${CLAUDE_SKILL_DIR}/../../scripts/hazard_lookup.py --lat {緯度} --lng {経度} --output /tmp/bcp_hazard.json
```

**3b. 地震リスク定量分析（J-SHIS API）:**
```bash
python3 ${CLAUDE_SKILL_DIR}/../../scripts/earthquake_lookup.py --lat {緯度} --lng {経度} --output /tmp/bcp_earthquake.json
```

**3d. ハザードマップ画像生成:**
```bash
python3 ${CLAUDE_SKILL_DIR}/../../scripts/generate_hazard_map.py --lat {緯度} --lng {経度} --output /tmp/bcp_hazard_map.png --base64
```

3dは国土地理院の標準地図に洪水・土砂・津波のハザードレイヤーを重ね合わせたPNG画像を生成する。
`--base64` フラグにより、Base64 data URIが `/tmp/bcp_hazard_map.png.b64.txt` にファイル保存される（標準出力にはファイルパスのみ出力）。
**⚠️ Base64ファイルは約1.2MBあるため、会話コンテキストには読み込まないこと。**
Step 6のHTML生成時にPythonスクリプトでファイルから読み込んでHTMLに埋め込む（後述）。

3aと3dはファイルキャッシュ（`/tmp/bcp_tile_cache/`）を共有し、タイルの二重取得を回避する。

### 並列グループB: WebSearch（グループAと同時に開始）

**3c. 広域リスク情報・自治体防災情報をWebSearchで収集**

以下を **並行で** WebSearch:
1. 「{自治体名} 地震被害想定 震度」
2. 「{自治体名} 地域防災計画」
3. 「{都道府県名} 南海トラフ 被害想定」（太平洋側の場合）
   または「{都道府県名} 首都直下 被害想定」（関東の場合）
4. 「{自治体名} ハザードマップ 防災マップ」
5. 「{自治体名} 防災ポータル 避難所一覧」

検索結果の上位ページで具体的なデータ（想定震度、被害者数、帰宅困難者等）があれば `WebFetch` で取得。

**自治体防災リンクの収集:**
- ハザードマップのURL（洪水、地震、土砂災害等）
- 防災ポータルサイトのURL
- 避難所一覧・マップのURL
- 地域防災計画のPDF/ページURL

→ これらのURLは Step 6 で付録「自治体防災リンク集」として掲載する。

**3e. 業界別BCP・防災情報の収集**

以下を **並行で** WebSearch:
1. 「{業種} 業界団体 BCP ガイドライン」
2. 「{業種} 協会 防災 事業継続」
3. 「{業種} 組合 災害対策 マニュアル」

**収集する情報:**
- 業界団体が公開しているBCPガイドライン・テンプレート
- 業界固有の災害リスク・対策事例
- 業界団体のBCP関連ページURL

**活用方法:**
- Step 4 のリスク判断で業界固有のリスク知見を参照
- Step 5 の重要業務・RTO設定で業界標準を参照
- Step 6 の対策セクションで業界推奨の具体策を反映
- 付録に「業界BCP参考資料」として出典・リンクを掲載

**見つからない場合:** 業界固有の情報がなくても中小企業庁BCPガイドラインで補完し、完走する。

### 各コマンドの出力データ

**3a出力** (JSON): 洪水・土砂災害・津波の4レイヤー分析結果

**3b出力** (JSON): 以下のデータを自動取得:
- **地震発生確率**: 30年以内に震度5弱/5強/6弱/6強以上となる確率（%）
- **表層地盤**: 微地形分類、AVS30（30m平均S波速度）、地盤増幅率、揺れやすさ
- **液状化リスク**: 微地形分類とAVS30に基づく簡易判定
- **想定震度**: 確率データから「発生しうる震度レンジ」を推定

**3d出力**: Base64 data URI（標準出力）+ PNG画像ファイル

## Step 4: 自律判断① リスク優先順位の決定

**ここが最重要。テンプレートでは不可能な、AIによる統合判断を行う。**

収集した全データを統合し、**この企業にとって**最も深刻なリスクTOP3を特定する。

判断の際に考慮すべき要素:
- ハザードマップの結果（浸水深、土砂リスク、津波リスク）
- **J-SHIS地震データ**: 震度別発生確率（%）、地盤増幅率、液状化リスク判定
- 地震の想定震度（J-SHISの定量データ + WebSearchの自治体被害想定を統合）
- 業種固有の脆弱性（例: 食品業×洪水→コールドチェーン断絶）
- 所在地の地理的特性（河川近接、海岸近接、山間部等）
- **地盤特性**: 微地形分類・AVS30・増幅率から揺れやすさを定量評価
- **業界団体のBCP情報**: Step 3eで収集した業界固有のリスク知見・対策事例

**出力形式（内部メモ）:**
```
リスク1: {リスク名} - 根拠: {ハザードデータ+業種分析}
リスク2: {リスク名} - 根拠: {同上}
リスク3: {リスク名} - 根拠: {同上}
```

リスクマトリクス（発生確率×影響度）も作成すること。

## Step 5: 自律判断② 重要業務とRTOの決定

`${CLAUDE_SKILL_DIR}/../../references/bcp-sections.md` の業種別ガイダンスを参照しつつ、
**この企業固有の**重要業務とRTOを決定する。

判断の際に考慮すべき要素:
- 企業の主要事業（Step 1で特定）
- 業種別のRTO基準（references参照）
- リスクTOP3に対する各業務の脆弱性
- サプライチェーン依存度

**出力形式（内部メモ）:**
```
重要業務1: {業務名} - RTO: {時間} - 根拠: {なぜこのRTOか}
重要業務2: {業務名} - RTO: {時間} - 根拠: {同上}
...
```

## Step 6: 自律判断③ BCP文書生成（HTML直接出力）

`${CLAUDE_SKILL_DIR}/../../references/bcp-sections.md` のセクション構造に従い、
`${CLAUDE_SKILL_DIR}/../../assets/bcp_template.css` のCSSを `<style>` タグで埋め込んだ
完全なHTMLファイルを生成する。

### 出力ファイル名の決定

ファイル名は企業名と日付から動的に生成する:
- **パターン**: `/tmp/bcp_{会社名}_{YYYYMMDD}.html`
- **例**: `/tmp/bcp_トヨタ自動車_20260321.html`
- **サニタイズ**: 会社名のスペースはアンダースコアに、ファイル名に使えない文字（`/\:*?"<>|`）は除去する
- **日付**: スキル実行日（`date +%Y%m%d` で取得）

以降、このファイルパスを `{出力パス}` と表記する。

### HTML生成ルール

1. **CSSファイルを事前に読み取る**: `Read` ツールで `${CLAUDE_SKILL_DIR}/../../assets/bcp_template.css` を読み、`<style>` タグ内にそのまま埋め込む

2. **セクション構成** (全セクション必須):
   - 表紙（class="cover-page"）
   - エグゼクティブサマリー（class="executive-summary"）
   - 目次（class="toc-page"）
   - 会社概要・事業概要
   - BCP基本方針
   - リスク分析（ハザードデータ含む、class="ai-analysis"で分析箇所をマーク）
   - 重要業務の特定
   - 目標復旧時間（RTO）表
   - 対策・行動計画（事前/発災直後/復旧の3フェーズ）
   - 緊急連絡体制（class="contact-table"で記入欄付き）
   - 備蓄・資源計画（class="checklist"でチェックリスト）
   - 付録（出典、免責事項）

3. **エグゼクティブサマリー**: 表紙の直後に `<div class="executive-summary">` で1ページのサマリーを生成する。経営者が最初の1ページで要点を掴めることが目的。以下の4カードで構成:
    ```html
    <div class="executive-summary">
        <h2>エグゼクティブサマリー</h2>
        <div class="summary-grid">
            <div class="summary-card overview">
                <h3>企業概要</h3>
                <p><strong>{会社名}</strong>（{業種}）</p>
                <p>所在地: {所在地}</p>
                <p>従業員数: {従業員数 or "要確認"}</p>
            </div>
            <div class="summary-card risks">
                <h3>主要リスク TOP3</h3>
                <ol>
                    <li>{リスク1名} <span class="risk-tag risk-tag-critical">深刻</span><br>{根拠要約}</li>
                    <li>{リスク2名} <span class="risk-tag risk-tag-high">高</span><br>{根拠要約}</li>
                    <li>{リスク3名} <span class="risk-tag risk-tag-medium">中</span><br>{根拠要約}</li>
                </ol>
            </div>
            <div class="summary-card operations">
                <h3>重要業務と目標復旧時間</h3>
                <ol>
                    <li><strong>{業務1}</strong> — RTO: {時間}</li>
                    <li><strong>{業務2}</strong> — RTO: {時間}</li>
                    <li><strong>{業務3}</strong> — RTO: {時間}</li>
                </ol>
            </div>
            <div class="summary-card actions">
                <h3>最優先アクション</h3>
                <ol>
                    <li>{最も重要な対策1}</li>
                    <li>{最も重要な対策2}</li>
                    <li>{最も重要な対策3}</li>
                </ol>
            </div>
        </div>
        <p class="summary-footer">※ 詳細は各セクションを参照してください。本文書は公開情報とAI分析に基づく参考資料です。</p>
    </div>
    ```
    - risk-tagのクラスはリスクの深刻度に応じて `risk-tag-critical`/`risk-tag-high`/`risk-tag-medium` を使い分ける
    - 各カードの内容はStep 4・5の判断結果をそのまま反映する

4. **業種×地域リスクに基づく具体的対策を記述すること**。汎用的な文言だけではNG。
   例: 「河川近接の食品製造業」なら「冷蔵設備の非常用電源確保（UPS + 発電機）」等

5. **`[要記入]` 欄**: 代表者名、BCP責任者、緊急連絡先の電話番号は `<span class="fill-in-inline">[要記入]</span>` とする

6. **リスクマトリクス**: `<div class="risk-matrix">` 内にtableで作成。セルにclass="risk-cell-critical/high/medium/low"を適用

7. **AI分析セクション**: Claudeが推論した箇所は `<div class="ai-analysis">` で囲む

8. **免責事項**: `${CLAUDE_SKILL_DIR}/../../references/sme-bcp-guidelines.md` の免責テンプレートを使用

9. **HTMLの文字コード**: `<meta charset="UTF-8">` を必ず含む

10. **改ページ制御**: 空の `<div class="page-break"></div>` は**絶対に使わないこと**（白紙ページが生成される）。セクション見出し `<h2 class="section-title">` に `page-break-before: always` がCSS側で設定済みなので、h2タグを置くだけで自動改ページされる

14. **テーブルの改ページ制御**: テーブルはデフォルトでページ跨ぎを許可し、ヘッダー行（`<thead>`）が次ページでも繰り返される。テーブルには必ず `<thead>` と `<tbody>` を使うこと。
    - **小さい表**（5行以下。連絡網、RTO表、企業概要など）: `<table class="no-break-table">` を使い、表全体を1ページに収める
    - **大きい表**（6行以上。リスク分析、チェックリスト、対策一覧など）: classなしの `<table>` を使い、ページ跨ぎを許可する

11. **ハザードマップ画像の埋め込み**: HTML生成時にBase64ファイルを直接読み込まないこと（約1.2MBありコンテキストを圧迫する）。代わりに以下の2段階で埋め込む:

    **Step 6a**: HTMLを生成する際、ハザードマップ部分にプレースホルダーを挿入する:
    ```html
    <div class="hazard-map-container">
        <img src="@@HAZARD_MAP_BASE64@@" alt="ハザードマップ">
        <p class="hazard-map-caption">図: 対象地点周辺のハザードマップ（赤丸が所在地）</p>
    </div>
    ```

    **Step 6b**: HTMLファイル出力後、Bashで以下を実行してプレースホルダーをBase64 data URIに置換する（`{出力パス}` は実際のファイルパスに置き換える）:
    ```bash
    python3 -c "
    import sys
    b64_path = '/tmp/bcp_hazard_map.png.b64.txt'
    html_path = '{出力パス}'
    try:
        with open(b64_path) as f:
            data_uri = f.read().strip()
        with open(html_path) as f:
            html = f.read()
        html = html.replace('@@HAZARD_MAP_BASE64@@', data_uri)
        with open(html_path, 'w') as f:
            f.write(html)
        print('ハザードマップ画像を埋め込みました')
    except FileNotFoundError:
        print('Base64ファイルが見つかりません。画像なしで続行します', file=sys.stderr)
    "
    ```

    `/tmp/bcp_hazard_map.png.b64.txt` が存在しない場合（Step 3d失敗時）は、プレースホルダーの代わりに代替テキストを入れておく:
    ```html
    <div class="hazard-map-container">
        <p class="hazard-map-caption">※ ハザードマップ画像は生成できませんでした。上記のハザード分析データを参照してください。</p>
    </div>
    ```

12. **自治体防災リンク集**: 付録セクションに Step 3c で収集した自治体の防災関連リンクを掲載する:
    ```html
    <h3>自治体防災情報リンク</h3>
    <div class="reference-links">
        <ul>
            <li><a href="{URL}">{自治体名} ハザードマップ</a></li>
            <li><a href="{URL}">{自治体名} 防災ポータル</a></li>
            <li><a href="{URL}">{自治体名} 避難所一覧</a></li>
            <li><a href="{URL}">{自治体名} 地域防災計画</a></li>
        </ul>
    </div>
    ```

13. **業界BCP参考資料**: 付録セクションに Step 3e で収集した業界団体のBCP関連情報を掲載する:
    ```html
    <h3>業界BCP参考資料</h3>
    <div class="reference-links">
        <ul>
            <li><a href="{URL}">{業界団体名} BCPガイドライン</a></li>
            <li>...</li>
        </ul>
        <p>※ 業界固有の情報が見つからなかった場合は省略可</p>
    </div>
    ```

## Step 7: ブラウザ表示＋結果報告

Step 6で `{出力パス}` に出力したHTMLが成果物。
ブラウザで開くだけで完全なBCP文書として閲覧可能。
印刷対応CSS付きのため、ブラウザの「印刷」（Cmd+P / Ctrl+P）→「PDFとして保存」で高品質なPDFも作成できる。

### 7a. HTMLをブラウザで自動表示

```bash
python3 -c "import webbrowser; webbrowser.open('file://{出力パス}')"
```

※ `{出力パス}` は実際のファイルパスに置き換える（例: `file:///tmp/bcp_トヨタ自動車_20260321.html`）。
※ ブラウザが開けない環境でもエラーにはならない。失敗した場合はファイルパスを報告して続行する。
