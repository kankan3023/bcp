# BCP自動生成スキル

企業WebサイトのURLからBCP（事業継続計画）を自動生成し、PDFとして出力します。

## スキルの動作

URLを受け取ったら、以下の7ステップを**自律的に**完走してください。
**ユーザーへの追加質問は原則ゼロ**。情報不足でも `[要確認]` で埋めて進行すること。

---

## Step 1: 情報収集（多重ソース）

`$ARGUMENTS` からURLを取得する。URLがなければユーザーに1回だけ聞く（唯一許容する質問）。

**以下を並行実行:**

1. `WebFetch` で企業サイトを読み取る
   - まずトップページ
   - 次に「会社概要」「about」「company」等のリンクを探してWebFetch
   - 「アクセス」「access」ページも探してWebFetch
2. サイトから会社名を特定したら `WebSearch` で「{会社名} 会社概要 事業内容 所在地」を検索

**抽出する情報:**
- 会社名
- 業種（推定含む）
- 事業内容・主要サービス/製品
- 所在地（住所）
- 従業員数（わかれば）
- 主要取引先（わかれば）

**自己回復ルール:**
- WebFetch失敗 → WebSearch結果のみで続行
- 住所不明 → WebSearchで追加検索「{会社名} 住所 アクセス」
- 最低限「業種の推定」と「住所の一部」があればStep 2へ進む
- 全情報源が失敗 → ユーザーに「業種と所在地」を1回だけ聞く

---

## Step 2: 住所→座標変換

```bash
python3 ${CLAUDE_SKILL_DIR}/../scripts/geocode.py "{住所}"
```

出力はJSON: `{"lat": 35.xxx, "lng": 139.xxx, "label": "..."}`

**フォールバック:**
- 失敗した場合、住所を簡略化して再試行（番地を除く等）
- それでも失敗 → WebSearchで「{会社名} 住所」を検索して再試行
- 最終手段: 都道府県庁所在地の座標で代替（`[所在地要確認]` 付記）

---

## Step 3: ハザードデータ収集

### 3a. ハザードマップタイル分析

```bash
python3 ${CLAUDE_SKILL_DIR}/../scripts/hazard_lookup.py --lat {緯度} --lng {経度} --output /tmp/bcp_hazard.json
```

出力はJSON。洪水・土砂災害・津波の4レイヤーを分析。

### 3b. 地震リスク定量分析（J-SHIS API）

```bash
python3 ${CLAUDE_SKILL_DIR}/../scripts/earthquake_lookup.py --lat {緯度} --lng {経度} --output /tmp/bcp_earthquake.json
```

出力はJSON。以下のデータを自動取得:
- **地震発生確率**: 30年以内に震度5弱/5強/6弱/6強以上となる確率（%）
- **表層地盤**: 微地形分類、AVS30（30m平均S波速度）、地盤増幅率、揺れやすさ
- **液状化リスク**: 微地形分類とAVS30に基づく簡易判定
- **想定震度**: 確率データから「発生しうる震度レンジ」を推定

**3aと3bは並行実行すること。**

### 3c. 広域リスク情報をWebSearchで収集

以下を **並行で** WebSearch:
1. 「{自治体名} 地震被害想定 震度」
2. 「{自治体名} 地域防災計画」
3. 「{都道府県名} 南海トラフ 被害想定」（太平洋側の場合）
   または「{都道府県名} 首都直下 被害想定」（関東の場合）

検索結果の上位ページで具体的なデータ（想定震度、被害者数、帰宅困難者等）があれば `WebFetch` で取得。

---

## Step 4: 自律判断① リスク優先順位の決定

**ここが最重要。テンプレートでは不可能な、AIによる統合判断を行う。**

収集した全データを統合し、**この企業にとって**最も深刻なリスクTOP3を特定する。

判断の際に考慮すべき要素:
- ハザードマップの結果（浸水深、土砂リスク、津波リスク）
- **J-SHIS地震データ**: 震度別発生確率（%）、地盤増幅率、液状化リスク判定
- 地震の想定震度（J-SHISの定量データ + WebSearchの自治体被害想定を統合）
- 業種固有の脆弱性（例: 食品業×洪水→コールドチェーン断絶）
- 所在地の地理的特性（河川近接、海岸近接、山間部等）
- **地盤特性**: 微地形分類・AVS30・増幅率から揺れやすさを定量評価

**出力形式（内部メモ）:**
```
リスク1: {リスク名} - 根拠: {ハザードデータ+業種分析}
リスク2: {リスク名} - 根拠: {同上}
リスク3: {リスク名} - 根拠: {同上}
```

リスクマトリクス（発生確率×影響度）も作成すること。

---

## Step 5: 自律判断② 重要業務とRTOの決定

`${CLAUDE_SKILL_DIR}/../references/bcp-sections.md` の業種別ガイダンスを参照しつつ、
**この企業固有の**重要業務とRTOを決定する。

判断の際に考慮すべき要素:
- 企業の主要事業（Step 1で特定）
- 業種別のRTO基準（references参照）
- リスクTOP3に対する各業務の脆弱性
- サプライチェーン依存度

**出力形式（内部メモ）:**
```
重要業務1: {業務名} - RTO: {時間} - 根拠: {なぜこのRTOか}
重要業務2: {業務名} - RTO: {時間} - 根拠: {同上}
...
```

---

## Step 6: 自律判断③ BCP文書生成（HTML直接出力）

`${CLAUDE_SKILL_DIR}/../references/bcp-sections.md` のセクション構造に従い、
`${CLAUDE_SKILL_DIR}/../assets/bcp_template.css` のCSSを `<style>` タグで埋め込んだ
完全なHTMLファイルを生成し、`/tmp/bcp_output.html` に出力する。

### HTML生成ルール

1. **CSSファイルを事前に読み取る**: `Read` ツールで `${CLAUDE_SKILL_DIR}/../assets/bcp_template.css` を読み、`<style>` タグ内にそのまま埋め込む

2. **セクション構成** (全セクション必須):
   - 表紙（class="cover-page"）
   - 目次（class="toc-page"）
   - 会社概要・事業概要
   - BCP基本方針
   - リスク分析（ハザードデータ含む、class="ai-analysis"で分析箇所をマーク）
   - 重要業務の特定
   - 目標復旧時間（RTO）表
   - 対策・行動計画（事前/発災直後/復旧の3フェーズ）
   - 緊急連絡体制（class="contact-table"で記入欄付き）
   - 備蓄・資源計画（class="checklist"でチェックリスト）
   - 付録（出典、免責事項）

3. **業種×地域リスクに基づく具体的対策を記述すること**。汎用的な文言だけではNG。
   例: 「河川近接の食品製造業」なら「冷蔵設備の非常用電源確保（UPS + 発電機）」等

4. **`[要記入]` 欄**: 代表者名、BCP責任者、緊急連絡先の電話番号は `<span class="fill-in-inline">[要記入]</span>` とする

5. **リスクマトリクス**: `<div class="risk-matrix">` 内にtableで作成。セルにclass="risk-cell-critical/high/medium/low"を適用

6. **AI分析セクション**: Claudeが推論した箇所は `<div class="ai-analysis">` で囲む

7. **免責事項**: `${CLAUDE_SKILL_DIR}/../references/sme-bcp-guidelines.md` の免責テンプレートを使用

8. **HTMLの文字コード**: `<meta charset="UTF-8">` を必ず含む

### Write ツールで出力
```
Write /tmp/bcp_output.html に完全なHTMLを出力
```

---

## Step 7: PDF生成＋結果報告

### 7a. PDF変換

```bash
python3 ${CLAUDE_SKILL_DIR}/../scripts/html_to_pdf.py /tmp/bcp_output.html /tmp/BCP_{会社名}_{YYYYMMDD}.pdf
```

会社名は英数字・ひらがな・カタカナ・漢字可。日付は生成日。

### 7b. ユーザーへの報告

PDF生成完了後、以下をユーザーに報告:

```
## BCP生成完了

**対象企業**: {会社名}
**出力ファイル**: /tmp/BCP_{会社名}_{YYYYMMDD}.pdf

### リスク分析サマリー
1. **{リスク1}**: {根拠の要約}
2. **{リスク2}**: {根拠の要約}
3. **{リスク3}**: {根拠の要約}

### 推奨アクション
- {最も優先度の高い対策3つ}

### 注意事項
- 本BCPは公開情報とAI分析に基づく参考資料です
- `[要記入]` 欄を実際の情報で埋めてください
- 年1回以上の見直しを推奨します
- 詳細は専門家にご相談ください
```

## エラーハンドリング（自己回復優先）

| 障害 | 対応 | ユーザーへの質問 |
|------|------|----------------|
| WebFetch失敗 | WebSearch結果で代替 | なし |
| 住所不明 | WebSearch追加検索 → 地域推定 | なし（`[要確認]`付記） |
| ジオコーディング失敗 | 都道府県レベルで続行 | なし |
| ハザードタイル取得失敗 | WebSearch結果のみで分析 | なし |
| WeasyPrint未インストール | スクリプト内で自動pip install | なし |
| PDF変換失敗 | HTMLファイルのパスを報告して続行 | なし |
| **全情報源が失敗** | 業種と所在地を1回聞いて汎用BCP生成 | **1回のみ** |

**原則: ユーザーへの質問はURL入力時の1回のみ。以降は完走する。**
