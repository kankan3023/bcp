# BCP策定案 自動生成スキル

**社名を入れるだけで、その企業専用のBCP（事業継続計画）の策定案が出てくる** Claude Codeスキルです。

BCP（事業継続計画）は、地震・洪水などの災害時に企業が事業を継続・早期復旧するための計画書です。策定には専門知識と多大な労力が必要で、中小企業の策定率はわずか17.1%にとどまっています（2025年 帝国データバンク調査）。このスキルは、公的ハザードデータ（J-SHIS地震・国土交通省洪水/土砂/津波）と業界BCP情報をAIが統合分析し、企業ごとにカスタマイズされたBCP策定案を自動生成します。

> AGI Lab AIエージェント ハッカソン 2026 出展作品

## クイックスタート

> 前提: [Claude Code](https://claude.ai/code) + Python 3.10以上

```bash
# 1. マーケットプレイスを追加
claude plugin marketplace add kankan3023/bcp

# 2. プラグインをインストール
claude plugin install bcp-generator@bcp-generator-marketplace

# 3. Claude Codeを再起動して実行
/bcp-generator:bcp トヨタ自動車
```

出力: `/tmp/bcp_output.html`（ブラウザで開けます）

社名の代わりにURLでもOK:
```bash
/bcp-generator:bcp https://example-company.co.jp
```

依存パッケージが自動インストールされない場合:
```bash
pip install -r plugins/bcp-generator/requirements.txt
```

## 特徴

- **社名でもURLでもOK**: 社名を入力すると自動で企業を特定。同名企業がある場合は候補リストを提示
- **公的データに基づくリスク分析**: J-SHIS地震ハザードデータ（震度別発生確率・地盤特性・液状化リスク）＋国土交通省ハザードマップ（洪水・土砂・津波）を自動取得
- **ハザードマップ画像を自動生成**: 国土地理院の地図にハザードレイヤーを重ね合わせた画像を文書に埋め込み
- **業種×地域に特化**: 汎用テンプレートではなく、業界団体のBCP情報も調査し企業固有のリスクと対策を提示
- **自治体防災情報を統合**: 所在市町村のハザードマップ・防災ポータル・避難所一覧へのリンクを付録に掲載
- **中小企業庁ガイドライン準拠**: BCPのセクション構成は公式指針に準拠
- **HTML出力**: ブラウザ表示＋印刷対応（Cmd+P → PDF保存可能）、明朝＋ゴシックの二書体構成（Mac/Win/Linux対応フォールバック）
- **追加質問ゼロで完走**: エラーが起きても自己回復して最後まで生成

```

    ├── 3b. J-SHIS地震リスク定量分析
    ├── 3c. 広域リスク＋自治体防災情報
    ├── 3d. ハザードマップ画像生成
    └── 3e. 業界BCP・防災情報収集
    │
    ▼
Step 4: 【AI自律判断①】リスク優先順位の決定
    │
    ▼
Step 5: 【AI自律判断②】重要業務とRTOの決定
    │
    ▼
Step 6: 【AI自律判断③】BCP文書生成（HTML）
    │
    ▼

```

## BCPの構成（全11セクション）

1. **表紙** - 会社名、作成日、バージョン
2. **目次**
3. **会社概要・事業概要** - Webサイトから自動収集
4. **BCP基本方針** - 目的、適用範囲、推進体制
5. **リスク分析** - ハザードマップ画像＋地震データ表＋リスクマトリクス＋TOP3リスク
6. **重要業務の特定** - AI分析による業務優先順位とボトルネック資源
7. **目標復旧時間（RTO）** - 業種×リスクに基づく設定、RLO含む
8. **対策・行動計画** - 事前/発災直後/復旧の3フェーズ
9. **緊急連絡体制** - 記入欄付きテンプレート＋外部関係機関
10. **備蓄・資源計画** - チェックリスト（業種固有の備蓄含む）
11. **付録** - データ出典、自治体防災リンク集、業界BCP参考資料、免責事項

## 技術構成

| コンポーネント | 技術・データソース |
|---|---|
| ジオコーディング | 国土地理院API + Nominatim（OSM） |
| ハザード分析 | 国土交通省ハザードマップタイル（洪水・土砂災害・津波 4レイヤー） |
| 地震リスク | 防災科学技術研究所 J-SHIS API（震度別確率・地盤特性・液状化） |
| ハザードマップ画像 | 国土地理院標準地図 + ハザードタイル重ね合わせ（Pillow） |
| 文書出力 | HTML（ブラウザ表示・印刷対応CSS付き） |
| フォント | 明朝＋ゴシック二書体（Mac: ヒラギノ / Win: 游明朝+BIZ UD / Linux: Noto CJK） |
| AI分析 | Claude による業種×地域×業界情報の統合判断（3つの自律判断ポイント） |

## データソースと根拠

| データ | 出典 | 取得方法 |
|---|---|---|
| 地震発生確率（震度別%） | 防災科学技術研究所 J-SHIS 2024年版 | APIで自動取得 |
| 表層地盤（AVS30・増幅率） | 防災科学技術研究所 J-SHIS V4 | APIで自動取得 |
| 洪水・土砂・津波リスク | 国土交通省 ハザードマップポータル | タイル画像を解析 |
| 企業情報 | 企業Webサイト + WebSearch | WebFetch/WebSearch |
| 広域被害想定 | 自治体防災計画・中央防災会議 | WebSearch |
| 業界BCP情報 | 業界団体公開資料 | WebSearch |

## 自律判断ポイント（審査基準: 自律性40%）

このスキルは以下の場面でAIが**自律的に**判断を行います:

1. **リスク優先順位の決定** - J-SHIS定量データ＋ハザードマップ＋業界情報＋業種脆弱性を統合し、企業固有のリスクTOP3を特定
2. **重要業務とRTOの決定** - 業種特性・サプライチェーン依存度・業界標準を考慮した復旧目標の設定
3. **BCP文書生成** - 業種×地域リスクに基づく具体的対策を含む完全なBCP文書（汎用文言ではなくカスタマイズされた内容）

### 自己回復（エラー耐性）

| 障害 | 対応 |
|---|---|
| WebFetch失敗 | WebSearch結果で代替 |
| 住所不明 | WebSearch追加検索→地域推定 |
| ジオコーディング失敗 | 都道府県レベルで続行 |
| ハザードタイル取得失敗 | WebSearch結果のみで分析 |
| 業界BCP情報なし | 中小企業庁ガイドラインで補完 |
| 全情報源が失敗 | ユーザーに業種と所在地を1回だけ聞いて生成 |
